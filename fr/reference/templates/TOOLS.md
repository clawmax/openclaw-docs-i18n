# Modèle TOOLS.md

Les compétences définissent *comment* les outils fonctionnent. Ce fichier est pour *vos* spécificités — les éléments uniques à votre configuration.

## Ce qui va ici

Des choses comme :

-   Les noms et emplacements des caméras
-   Les hôtes et alias SSH
-   Les voix préférées pour la TTS
-   Les noms des enceintes/pièces
-   Les surnoms des appareils
-   Tout ce qui est spécifique à l'environnement

## Exemples

```markdown
### Cameras

- living-room → Pièce principale, angle large 180°
- front-door → Entrée, déclenchement par mouvement

### SSH

- home-server → 192.168.1.100, utilisateur : admin

### TTS

- Voix préférée : "Nova" (chaude, légèrement britannique)
- Enceinte par défaut : HomePod de la cuisine
```

## Pourquoi les séparer ?

Les compétences sont partagées. Votre configuration vous appartient. Les garder séparées signifie que vous pouvez mettre à jour les compétences sans perdre vos notes, et partager les compétences sans divulguer votre infrastructure.
