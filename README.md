## 项目说明 / Project Description

### 中文

本仓库是 **OpenClaw 官方文档的国际化翻译版本**，面向全球用户提供多语言文档，让不同国家和地区的用户能够用**本国语言**阅读、理解和使用 OpenClaw。

**核心目标：**

- **本土化阅读**：将 OpenClaw 官方文档从英文翻译为多种语言，降低语言门槛，方便非英语用户上手。
- **覆盖多语种**：当前支持英语（en）、中文（zh）、日语（ja）、法语（fr）、俄语（ru）、西班牙语（es）、阿拉伯语（ar），阿拉伯语支持 RTL 布局。
- **与官网一致**：在结构、导航和阅读体验上对齐官方文档站，仅增加语言切换与多语言内容，便于全球用户按自己语言查找文档。

**面向谁：** 希望用母语阅读 OpenClaw 文档的用户；需要为本地社区或客户提供本地化文档的布道师、集成商或社区维护者；关心 OpenClaw 文档可及性与国际化的贡献者与维护者。

### English

This repository is the **internationalized translation of the official OpenClaw documentation**. It provides multilingual documentation so that users around the world can read, understand, and use OpenClaw in **their own language**.

**Goals:**

- **Localized reading**: Translate the official OpenClaw docs from English into multiple languages to lower the language barrier and make it easier for non-English users to get started.
- **Multiple languages**: Currently supports English (en), Chinese (zh), Japanese (ja), French (fr), Russian (ru), Spanish (es), and Arabic (ar), with RTL layout for Arabic.
- **Aligned with the official site**: Matches the structure, navigation, and reading experience of the official docs, with added language switching and multilingual content so users can find documentation in their language.

**For whom:** Users who want to read OpenClaw docs in their native language; advocates, integrators, or community maintainers who need localized docs for their region or customers; contributors and maintainers who care about documentation accessibility and internationalization.

## 技术栈

- **Next.js 15** (App Router)
- **MDX** (@next/mdx、next-mdx-remote)
- **remark-gfm**：GFM（表格、删除线、任务列表等）
- **rehype-slug**：为标题自动生成 `id`，用于右侧目录锚点
- **Tailwind CSS**：样式

## 国际化（i18n）

- **支持语言**：英语 (en)、中文 (zh)、日语 (ja)、法语 (fr)、俄语 (ru)、西班牙语 (es)、阿拉伯语 (ar)。阿拉伯语为 RTL 布局。
- **路由**：所有文档在 `/[lang]/` 下，如 `/en`、`/zh`。访问 `/` 会重定向到 `/en`。
- **导航可配置**：顶部 Navbar 与左侧 Sidebar 的文案均按语言配置，切换语言后导航文字随之切换。
- **配置位置**：
  - **导航**：`lib/nav.config.tsx`，按 locale 配置 `navbar`、`sidebar`、`onThisPage`、`languageLabel`。
  - **侧栏条目标题**：`lib/nav.sidebar-item-titles.ts`，各语言侧栏链接标题映射。
  - **内容**：`content/[lang]/`，每种语言对应目录；缺省时回退到 `content/en/`。
- **语言切换**：Navbar 右侧下拉选择语言，跳转到对应 `/[lang]`，整站导航与内容均切换为该语言。

## 布局与组件

- **Navbar**：顶部固定，Logo + 可配置外链 + 语言切换下拉
- **Sidebar**：左侧固定导航，来自 `navConfig.sidebar`，当前路径高亮，链接带 `/[lang]/` 前缀
- **主内容区**：MDX 正文（由 `content/[lang]/...` 渲染），站内链接通过 `LocaleLink` 自动加 locale 前缀
- **右侧 TOC**：根据当前页 h1/h2/h3 生成「On this page」，带滚动高亮

## 运行

```bash
cd docs-next
npm install
npm run dev
```

访问 http://localhost:3000（若 3000 被占用会使用 3001）。根路径会重定向到 `/en`。

## 导出为 Markdown 项目

将站点内容导出为**纯 Markdown**（无 MDX/JSX），含图片，便于上传到 GitHub。请在**仓库根目录**执行：

```bash
npm run export-markdown
```

默认输出到根目录下 `docs-export/`，结构：`en/`、`zh/`、… 与网站一致，`images/` 为图片。指定输出目录：

```bash
npm run export-markdown -- path/to/output
```

## 批量翻译文档

将 `content/en` 下的英文 MDX 翻译到 zh/ja/fr/ru/es/ar 使用项目根目录的脚本：

```bash
# 从项目根执行（需配置 DEEPSEEK_API_KEY 或 OPENAI_API_KEY 等）
node scripts/translate-docs.mjs --dry-run                    # 仅列出待翻译文件
node scripts/translate-docs.mjs --lang=zh --limit=20         # 仅翻译到中文，每种语言最多 20 篇
node scripts/translate-docs.mjs --slug=install               # 只翻译 install.mdx 到所有语言
node scripts/translate-docs.mjs                              # 翻译全部缺失文件（建议分批）
```

脚本会保持代码块、图片路径、内部链接与 JSX 不变，仅翻译 frontmatter 与正文可见文案。详见 `scripts/translate-docs.mjs` 顶部注释。
