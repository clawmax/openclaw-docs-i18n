# ClawHub

ClawHubは、**OpenClawのための公開スキルレジストリ**です。これは無料サービスです：すべてのスキルは公開されており、誰でも共有や再利用のために閲覧できます。スキルは、`SKILL.md`ファイル（および補助的なテキストファイル）を含む単なるフォルダです。Webアプリでスキルを閲覧したり、CLIを使用してスキルを検索、インストール、更新、公開したりできます。サイト: [clawhub.ai](https://clawhub.ai)

## ClawHubとは

-   OpenClawスキルのための公開レジストリ。
-   スキルバンドルとメタデータのバージョン管理されたストア。
-   検索、タグ、使用状況シグナルのための発見インターフェース。

## 仕組み

1.  ユーザーがスキルバンドル（ファイル＋メタデータ）を公開します。
2.  ClawHubがバンドルを保存し、メタデータを解析し、バージョンを割り当てます。
3.  レジストリがスキルを検索と発見のためにインデックス化します。
4.  ユーザーがスキルを閲覧、ダウンロードし、OpenClawにインストールします。

## できること

-   新しいスキルや既存スキルの新しいバージョンを公開する。
-   名前、タグ、または検索でスキルを発見する。
-   スキルバンドルをダウンロードし、そのファイルを検査する。
-   不正または安全でないスキルを報告する。
-   モデレーターの場合は、非表示、表示、削除、または禁止を行う。

## 対象ユーザー（初心者向け）

OpenClawエージェントに新しい機能を追加したい場合、ClawHubはスキルを見つけてインストールする最も簡単な方法です。バックエンドの仕組みを知る必要はありません。以下のことができます：

-   平易な言語でスキルを検索する。
-   ワークスペースにスキルをインストールする。
-   後で1つのコマンドでスキルを更新する。
-   独自のスキルを公開してバックアップする。

## クイックスタート（非技術者向け）

1.  CLIをインストールします（次のセクションを参照）。
2.  必要なものを検索します：
    -   `clawhub search "calendar"`
3.  スキルをインストールします：
    -   `clawhub install <skill-slug>`
4.  新しいOpenClawセッションを開始して、新しいスキルを認識させます。

## CLIのインストール

```bash

```

## OpenClawとの連携

デフォルトでは、CLIは現在の作業ディレクトリの下の`./skills`にスキルをインストールします。OpenClawワークスペースが設定されている場合、`clawhub`は`--workdir`（または`CLAWHUB_WORKDIR`）で上書きしない限り、そのワークスペースにフォールバックします。OpenClawはワークスペーススキルを`/skills`から読み込み、**次の**セッションでそれらを認識します。すでに`~/.openclaw/skills`やバンドルされたスキルを使用している場合、ワークスペーススキルが優先されます。スキルがどのように読み込まれ、共有され、ゲートされるかの詳細については、[スキル](./skills.md)を参照してください。

## スキルシステム概要

スキルは、OpenClawに特定のタスクを実行する方法を教えるバージョン管理されたファイルのバンドルです。各公開は新しいバージョンを作成し、レジストリはバージョンの履歴を保持するため、ユーザーは変更を監査できます。典型的なスキルには以下が含まれます：

-   主要な説明と使用方法を含む`SKILL.md`ファイル。
-   スキルで使用されるオプションの設定、スクリプト、または補助ファイル。
-   タグ、要約、インストール要件などのメタデータ。

ClawHubはメタデータを使用して発見を強化し、スキルの機能を安全に公開します。レジストリはまた、スキルのランキングと可視性を向上させるために、使用状況シグナル（スターやダウンロード数など）を追跡します。

## サービスが提供するもの（機能）

-   スキルとその`SKILL.md`コンテンツの**公開閲覧**。
-   キーワードだけでなく、埋め込み（ベクター検索）による**検索**。
-   semver、変更履歴、タグ（`latest`を含む）による**バージョン管理**。
-   バージョンごとのzipとしての**ダウンロード**。
-   コミュニティフィードバックのための**スターとコメント**。
-   承認と監査のための**モデレーション**フック。
-   自動化とスクリプティングのための**CLIフレンドリーなAPI**。

## セキュリティとモデレーション

ClawHubはデフォルトでオープンです。誰でもスキルをアップロードできますが、公開するにはGitHubアカウントが少なくとも1週間経過している必要があります。これは、正当な貢献者をブロックすることなく、悪用を遅らせるのに役立ちます。報告とモデレーション：

-   サインインしているユーザーは誰でもスキルを報告できます。
-   報告理由は必須で記録されます。
-   各ユーザーは最大20件のアクティブな報告を同時に持つことができます。
-   3件以上のユニークな報告があるスキルは、デフォルトで自動的に非表示になります。
-   モデレーターは非表示のスキルを表示し、表示を戻し、削除し、またはユーザーを禁止できます。
-   報告機能の悪用はアカウントの禁止につながる可能性があります。

モデレーターになりたいですか？OpenClaw Discordで質問し、モデレーターまたはメンテナーに連絡してください。

## CLIコマンドとパラメータ

グローバルオプション（すべてのコマンドに適用）：

-   `--workdir `: 作業ディレクトリ（デフォルト：現在のディレクトリ；OpenClawワークスペースにフォールバック）。
-   `--dir `: スキルディレクトリ、workdirからの相対パス（デフォルト：`skills`）。
-   `--site `: サイトベースURL（ブラウザーログイン）。
-   `--registry `: レジストリAPIベースURL。
-   `--no-input`: プロンプトを無効化（非対話型）。
-   `-V, --cli-version`: CLIバージョンを表示。

認証：

-   `clawhub login`（ブラウザーフロー）または`clawhub login --token `
-   `clawhub logout`
-   `clawhub whoami`

オプション：

-   `--token `: APIトークンを貼り付け。
-   `--label `: ブラウザーログイントークンに保存されるラベル（デフォルト：`CLI token`）。
-   `--no-browser`: ブラウザーを開かない（`--token`が必要）。

検索：

-   `clawhub search "query"`
-   `--limit `: 最大結果数。

インストール：

-   `clawhub install `
-   `--version `: 特定のバージョンをインストール。
-   `--force`: フォルダが既に存在する場合に上書き。

更新：

-   `clawhub update `
-   `clawhub update --all`
-   `--version `: 特定のバージョンに更新（単一スラッグのみ）。
-   `--force`: ローカルファイルが公開バージョンと一致しない場合に上書き。

リスト：

-   `clawhub list` (`.clawhub/lock.json`を読み取る)

公開：

-   `clawhub publish `
-   `--slug `: スキルスラッグ。
-   `--name `: 表示名。
-   `--version `: Semverバージョン。
-   `--changelog `: 変更履歴テキスト（空でも可）。
-   `--tags `: カンマ区切りのタグ（デフォルト：`latest`）。

削除/削除解除（所有者/管理者のみ）：

-   `clawhub delete  --yes`
-   `clawhub undelete  --yes`

同期（ローカルスキルのスキャン＋新規/更新されたものを公開）：

-   `clawhub sync`
-   `--root <dir...>`: 追加のスキャンルート。
-   `--all`: プロンプトなしですべてをアップロード。
-   `--dry-run`: アップロードされる内容を表示。
-   `--bump `: 更新用の`patch|minor|major`（デフォルト：`patch`）。
-   `--changelog `: 非対話型更新のための変更履歴。
-   `--tags `: カンマ区切りのタグ（デフォルト：`latest`）。
-   `--concurrency `: レジストリチェック（デフォルト：4）。

## エージェントのための一般的なワークフロー

### スキルの検索

```bash
clawhub search "postgres backups"
```

### 新しいスキルのダウンロード

```bash
clawhub install my-skill-pack
```

### インストール済みスキルの更新

```bash
clawhub update --all
```

### スキルのバックアップ（公開または同期）

単一のスキルフォルダの場合：

```bash
clawhub publish ./my-skill --slug my-skill --name "My Skill" --version 1.0.0 --tags latest
```

一度に多くのスキルをスキャンしてバックアップする場合：

```bash
clawhub sync --all
```

## 高度な詳細（技術的）

### バージョン管理とタグ

-   各公開は新しい**semver** `SkillVersion`を作成します。
-   タグ（`latest`など）はバージョンを指し、タグを移動することでロールバックが可能です。
-   変更履歴はバージョンごとに添付され、同期や更新を公開する際は空でも構いません。

### ローカル変更とレジストリバージョン

更新は、ローカルのスキル内容とレジストリバージョンをコンテンツハッシュを使用して比較します。ローカルファイルが公開バージョンと一致しない場合、CLIは上書きする前に確認を求めます（非対話型実行では`--force`が必要です）。

### 同期スキャンとフォールバックルート

`clawhub sync`は最初に現在のworkdirをスキャンします。スキルが見つからない場合、既知のレガシー場所（例：`~/openclaw/skills`や`~/.openclaw/skills`）にフォールバックします。これは、追加のフラグなしで古いスキルインストールを見つけるように設計されています。

### ストレージとロックファイル

-   インストールされたスキルは、workdirの下の`.clawhub/lock.json`に記録されます。
-   認証トークンはClawHub CLI設定ファイルに保存されます（`CLAWHUB_CONFIG_PATH`で上書き可能）。

### テレメトリー（インストール数）

ログイン状態で`clawhub sync`を実行すると、CLIはインストール数を計算するための最小限のスナップショットを送信します。これは完全に無効化できます：

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

## 環境変数

-   `CLAWHUB_SITE`: サイトURLを上書き。
-   `CLAWHUB_REGISTRY`: レジストリAPI URLを上書き。
-   `CLAWHUB_CONFIG_PATH`: CLIがトークン/設定を保存する場所を上書き。
-   `CLAWHUB_WORKDIR`: デフォルトのworkdirを上書き。
-   `CLAWHUB_DISABLE_TELEMETRY=1`: `sync`時のテレメトリーを無効化。
