# ブラウザログイン

## 手動ログイン（推奨）

サイトがログインを要求する場合、**ホスト**ブラウザプロファイル（openclawブラウザ）で**手動でサインイン**してください。モデルに認証情報を**与えないでください**。自動化されたログインはアンチボット対策を頻繁にトリガーし、アカウントをロックする可能性があります。メインのブラウザドキュメントに戻る: [ブラウザ](./browser.md)。

## どのChromeプロファイルが使用されますか？

OpenClawは**専用のChromeプロファイル**（`openclaw`という名前、オレンジ色のUI）を制御します。これは日常使用するブラウザプロファイルとは別物です。アクセスする簡単な方法は2つあります:

1.  **エージェントにブラウザを開くよう依頼し**、その後自分でログインする。
2.  **CLI経由で開く**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

複数のプロファイルがある場合は、`--browser-profile `を渡してください（デフォルトは`openclaw`です）。

## X/Twitter: 推奨フロー

-   **閲覧/検索/スレッド:** **ホスト**ブラウザ（手動ログイン）を使用する。
-   **投稿更新:** **ホスト**ブラウザ（手動ログイン）を使用する。

## サンドボックス化 + ホストブラウザアクセス

サンドボックス化されたブラウザセッションは、ボット検知を**より引き起こしやすい**です。X/Twitter（およびその他の厳格なサイト）では、**ホスト**ブラウザを優先してください。エージェントがサンドボックス化されている場合、ブラウザツールはデフォルトでサンドボックスを使用します。ホスト制御を許可するには:

```json
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

その後、ホストブラウザをターゲットにします:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

または、投稿更新を行うエージェントのサンドボックス化を無効にします。
