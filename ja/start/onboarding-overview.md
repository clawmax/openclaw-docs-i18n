# オンボーディング概要

OpenClawは、Gatewayの実行場所やプロバイダーの設定方法に応じて、複数のオンボーディングパスをサポートしています。

## オンボーディングパスを選択

-   **CLIウィザード** - macOS、Linux、Windows（WSL2経由）用。
-   **macOSアプリ** - AppleシリコンまたはIntel Macでのガイド付き初回実行用。

## CLIオンボーディングウィザード

ターミナルでウィザードを実行します：

```bash
openclaw onboard
```

Gateway、ワークスペース、チャネル、スキルを完全に制御したい場合は、CLIウィザードを使用してください。関連ドキュメント：

-   [オンボーディングウィザード (CLI)](./wizard.md)
-   [`openclaw onboard` コマンド](../cli/onboard.md)

## macOSアプリオンボーディング

macOSで完全にガイドされたセットアップを希望する場合は、OpenClawアプリを使用してください。関連ドキュメント：

-   [オンボーディング (macOSアプリ)](./onboarding.md)

## カスタムプロバイダー

リストにないエンドポイント（標準のOpenAIまたはAnthropic APIを公開するホスト型プロバイダーを含む）が必要な場合は、CLIウィザードで**カスタムプロバイダー**を選択してください。以下の情報を求められます：

-   OpenAI互換、Anthropic互換、または**不明**（自動検出）を選択。
-   ベースURLとAPIキー（プロバイダーで必要な場合）を入力。
-   モデルIDとオプションのエイリアスを提供。
-   複数のカスタムエンドポイントが共存できるように、エンドポイントIDを選択。

詳細な手順については、上記のCLIオンボーディングドキュメントに従ってください。
