# memory

Управление семантической индексацией памяти и поиском. Предоставляется активным плагином памяти (по умолчанию: `memory-core`; установите `plugins.slots.memory = "none"` для отключения). Связанные темы:

-   Концепция памяти: [Память](../concepts/memory.md)
-   Плагины: [Плагины](../tools/plugin.md)

## Примеры

```bash
openclaw memory status
openclaw memory status --deep
openclaw memory index --force
openclaw memory search "meeting notes"
openclaw memory search --query "deployment" --max-results 20
openclaw memory status --json
openclaw memory status --deep --index
openclaw memory status --deep --index --verbose
openclaw memory status --agent main
openclaw memory index --agent main --verbose
```

## Опции

`memory status` и `memory index`:

-   `--agent `: ограничить областью действия одним агентом. Без этого параметра команды выполняются для каждого настроенного агента; если список агентов не настроен, они возвращаются к агенту по умолчанию.
-   `--verbose`: выводить подробные логи во время проверок и индексации.

`memory status`:

-   `--deep`: проверить доступность векторного хранилища и эмбеддингов.
-   `--index`: выполнить переиндексацию, если хранилище помечено как "грязное" (подразумевает `--deep`).
-   `--json`: вывести вывод в формате JSON.

`memory index`:

-   `--force`: принудительно выполнить полную переиндексацию.

`memory search`:

-   Ввод запроса: передайте либо позиционный аргумент `[query]`, либо `--query `.
-   Если указаны оба, приоритет имеет `--query`.
-   Если не указан ни один, команда завершается с ошибкой.
-   `--agent `: ограничить областью действия одним агентом (по умолчанию: агент по умолчанию).
-   `--max-results `: ограничить количество возвращаемых результатов.
-   `--min-score `: отфильтровать совпадения с низким баллом.
-   `--json`: вывести результаты в формате JSON.

Примечания:

-   `memory index --verbose` выводит детали по каждой фазе (провайдер, модель, источники, активность пакетов).
-   `memory status` включает любые дополнительные пути, настроенные через `memorySearch.extraPaths`.
-   Если поля ключей API удалённой памяти, которые фактически активны, настроены как SecretRefs, команда разрешает эти значения из активного снимка состояния шлюза. Если шлюз недоступен, команда быстро завершается с ошибкой.
-   Примечание о версионном несоответствии шлюза: для этого пути команды требуется шлюз, поддерживающий `secrets.resolve`; более старые шлюзы возвращают ошибку неизвестного метода.
