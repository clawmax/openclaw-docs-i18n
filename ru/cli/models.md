# models

Обнаружение, сканирование и конфигурация моделей (модель по умолчанию, резервные модели, профили аутентификации). Связанные разделы:

-   Провайдеры + модели: [Модели](../providers/models.md)
-   Настройка аутентификации провайдера: [Начало работы](../start/getting-started.md)

## Часто используемые команды

```bash
openclaw models status
openclaw models list
openclaw models set <model-or-alias>
openclaw models scan
```

`openclaw models status` показывает разрешённые модели по умолчанию/резервные модели, а также обзор аутентификации. Когда доступны снимки использования провайдеров, раздел статуса OAuth/токена включает заголовки использования провайдера. Добавьте `--probe` для выполнения живых проверок аутентификации для каждого настроенного профиля провайдера. Проверки — это реальные запросы (могут расходовать токены и вызывать ограничения по частоте). Используйте `--agent ` для проверки настроенного состояния модели/аутентификации агента. Если не указано, команда использует `OPENCLAW_AGENT_DIR`/`PI_CODING_AGENT_DIR`, если они заданы, в противном случае — настроенного агента по умолчанию. Примечания:

-   `models set <model-or-alias>` принимает `provider/model` или алиас.
-   Ссылки на модели разбираются путём разделения по **первому** `/`. Если идентификатор модели содержит `/` (в стиле OpenRouter), укажите префикс провайдера (пример: `openrouter/moonshotai/kimi-k2`).
-   Если вы опустите провайдера, OpenClaw рассматривает ввод как алиас или модель для **провайдера по умолчанию** (работает только тогда, когда в идентификаторе модели нет `/`).
-   `models status` может показывать `marker()` в выводе аутентификации для несекретных заполнителей (например, `OPENAI_API_KEY`, `secretref-managed`, `minimax-oauth`, `qwen-oauth`, `ollama-local`) вместо маскировки их как секретов.

### models status

Опции:

-   `--json`
-   `--plain`
-   `--check` (выход 1=истёк/отсутствует, 2=истекает)
-   `--probe` (живая проверка настроенных профилей аутентификации)
-   `--probe-provider ` (проверить одного провайдера)
-   `--probe-profile ` (повторить или идентификаторы профилей через запятую)
-   `--probe-timeout `
-   `--probe-concurrency `
-   `--probe-max-tokens `
-   `--agent ` (настроенный идентификатор агента; переопределяет `OPENCLAW_AGENT_DIR`/`PI_CODING_AGENT_DIR`)

## Профили аутентификации

```bash
openclaw models auth add
openclaw models auth login --provider <id>
openclaw models auth setup-token
openclaw models auth paste-token
```

`models auth login` запускает процесс аутентификации плагина провайдера (OAuth/API-ключ). Используйте `openclaw plugins list`, чтобы увидеть, какие провайдеры установлены. Примечания:

-   `setup-token` запрашивает значение токена настройки (сгенерируйте его с помощью `claude setup-token` на любой машине).
-   `paste-token` принимает строку токена, сгенерированную в другом месте или с помощью автоматизации.
-   Примечание о политике Anthropic: поддержка токена настройки — это техническая совместимость. Anthropic ранее блокировал некоторое использование подписок вне Claude Code, поэтому проверяйте текущие условия перед широким использованием.
