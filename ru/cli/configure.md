# configure

Интерактивный мастер для настройки учетных данных, устройств и параметров агентов по умолчанию. Примечание: Раздел **Model** теперь включает множественный выбор для разрешенного списка `agents.defaults.models` (что отображается в `/model` и в выборе модели). Совет: `openclaw config` без подкоманды открывает тот же мастер. Используйте `openclaw config get|set|unset` для неинтерактивного редактирования. Связанные разделы:

-   Справочник по конфигурации шлюза: [Конфигурация](../gateway/configuration.md)
-   Команда config CLI: [Config](./config.md)

Примечания:

-   Выбор места запуска Шлюза всегда обновляет `gateway.mode`. Вы можете выбрать "Продолжить" без других разделов, если это все, что вам нужно.
-   Сервисы, ориентированные на каналы (Slack/Discord/Matrix/Microsoft Teams), запрашивают разрешенные списки каналов/комнат во время настройки. Вы можете ввести имена или ID; мастер преобразует имена в ID, когда это возможно.
-   Если вы выполняете шаг установки демона, аутентификация по токену требует токен, и `gateway.auth.token` управляется через SecretRef. Команда configure проверяет SecretRef, но не сохраняет расшифрованные значения обычного текста токена в метаданные среды службы супервизора.
-   Если для аутентификации по токену требуется токен, а настроенный SecretRef токена не разрешен, команда configure блокирует установку демона, предоставляя руководство по исправлению.
-   Если настроены и `gateway.auth.token`, и `gateway.auth.password`, а `gateway.auth.mode` не задан, команда configure блокирует установку демона до тех пор, пока режим не будет задан явно.

## Примеры

```bash
openclaw configure
openclaw configure --section model --section channels
```
