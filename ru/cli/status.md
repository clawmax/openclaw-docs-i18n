# status

Диагностика каналов и сессий.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

Примечания:

-   `--deep` выполняет живые проверки (WhatsApp Web + Telegram + Discord + Google Chat + Slack + Signal).
-   Вывод включает хранилища сессий для каждого агента, когда сконфигурировано несколько агентов.
-   Обзор включает статус установки/работы службы Gateway и узла, когда это доступно.
-   Обзор включает канал обновлений и git SHA (для сборок из исходного кода).
-   Информация об обновлениях отображается в Обзоре; если обновление доступно, status выводит подсказку выполнить `openclaw update` (см. [Обновление](../install/updating.md)).
-   Команды статуса только для чтения (`status`, `status --json`, `status --all`) разрешают поддерживаемые SecretRefs для их целевых путей конфигурации, когда это возможно.
-   Если поддерживаемый SecretRef канала сконфигурирован, но недоступен в текущем пути выполнения команды, статус остаётся режимом только для чтения и выводит ухудшенные данные вместо сбоя. Человекочитаемый вывод показывает предупреждения, такие как «сконфигурированный токен недоступен в этом пути выполнения команды», а вывод JSON включает `secretDiagnostics`.
