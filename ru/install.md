# Установка

Уже следовали [Началу работы](./start/getting-started.md)? Всё готово — эта страница предназначена для альтернативных методов установки, инструкций для конкретных платформ и обслуживания.

## Системные требования

-   **[Node 22+](./install/node.md)** ([скрипт установщика](#install-methods) установит его, если он отсутствует)
-   macOS, Linux или Windows
-   `pnpm` только если вы собираете из исходного кода

> **ℹ️** В Windows настоятельно рекомендуем запускать OpenClaw под [WSL2](https://learn.microsoft.com/en-us/windows/wsl/install).

## Методы установки

> **💡** **Скрипт установщика** — рекомендуемый способ установки OpenClaw. Он обрабатывает обнаружение Node, установку и первоначальную настройку за один шаг.

 

> **⚠️** Для VPS/облачных хостов по возможности избегайте сторонних образов из маркетплейсов «в 1 клик». Предпочтительнее чистый базовый образ ОС (например, Ubuntu LTS), затем установите OpenClaw самостоятельно с помощью скрипта установщика.

 

Скачивает CLI, устанавливает его глобально через npm и запускает мастер первоначальной настройки.

Всё — скрипт сам обрабатывает обнаружение Node, установку и первоначальную настройку.Чтобы пропустить первоначальную настройку и просто установить бинарный файл:

Все флаги, переменные окружения и опции для CI/автоматизации смотрите в [Внутреннее устройство установщика](./install/installer.md).

Если у вас уже есть Node 22+ и вы предпочитаете управлять установкой самостоятельно:

Для участников разработки или тех, кто хочет запускать из локальной копии репозитория.

1

[

](#)

Клонировать и собрать

Клонируйте [репозиторий OpenClaw](https://github.com/openclaw/openclaw) и соберите проект:

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm ui:build
pnpm build
```

2

[

](#)

Связать CLI

Сделайте команду `openclaw` доступной глобально:

```bash
pnpm link --global
```

В качестве альтернативы, пропустите связывание и запускайте команды через `pnpm openclaw ...` изнутри репозитория.

3

[

](#)

Запустить первоначальную настройку

```bash
openclaw onboard --install-daemon
```

Для более глубоких рабочих процессов разработки см. [Настройка](./start/setup.md).

## После установки

Проверьте, что всё работает:

```bash
openclaw doctor         # проверить наличие проблем с конфигурацией
openclaw status         # статус шлюза
openclaw dashboard      # открыть веб-интерфейс
```

Если вам нужны пользовательские пути для среды выполнения, используйте:

-   `OPENCLAW_HOME` для внутренних путей, основанных на домашнем каталоге
-   `OPENCLAW_STATE_DIR` для расположения изменяемого состояния
-   `OPENCLAW_CONFIG_PATH` для расположения файла конфигурации

См. [Переменные окружения](./help/environment.md) для порядка приоритета и полных деталей.

## Устранение неполадок: openclaw не найден

Быстрая диагностика:

```bash
node -v
npm -v
npm prefix -g
echo "$PATH"
```

Если `$(npm prefix -g)/bin` (macOS/Linux) или `$(npm prefix -g)` (Windows) **не** находится в вашем `$PATH`, ваша оболочка не может найти глобальные бинарные файлы npm (включая `openclaw`).Исправление — добавьте его в файл запуска вашей оболочки (`~/.zshrc` или `~/.bashrc`):

```bash
export PATH="$(npm prefix -g)/bin:$PATH"
```

В Windows добавьте вывод команды `npm prefix -g` в ваш PATH.Затем откройте новый терминал (или выполните `rehash` в zsh / `hash -r` в bash).
