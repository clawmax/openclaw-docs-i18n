# Agent Send

`openclaw agent` выполняет один ход агента без необходимости входящего сообщения чата. По умолчанию он работает **через Шлюз**; добавьте `--local`, чтобы принудительно использовать встроенную среду выполнения на текущей машине.

## Поведение

-   Обязательно: `--message <текст>`
-   Выбор сессии:
    -   `--to <адрес>` определяет ключ сессии (целевые группы/каналы сохраняют изоляцию; прямые чаты сводятся к `main`), **или**
    -   `--session-id ` повторно использует существующую сессию по id, **или**
    -   `--agent ` нацеливается на настроенного агента напрямую (использует ключ сессии `main` этого агента)
-   Запускает ту же встроенную среду выполнения агента, что и обычные входящие ответы.
-   Флаги мышления/подробного вывода сохраняются в хранилище сессий.
-   Вывод:
    -   по умолчанию: печатает текст ответа (плюс строки `MEDIA:`)
    -   `--json`: печатает структурированные данные + метаданные
-   Опциональная доставка обратно в канал с помощью `--deliver` + `--channel` (форматы цели соответствуют `openclaw message --target`).
-   Используйте `--reply-channel`/`--reply-to`/`--reply-account` для переопределения доставки без изменения сессии.

Если Шлюз недоступен, CLI **переключается** на встроенный локальный запуск.

## Примеры

```bash
openclaw agent --to +15555550123 --message "status update"
openclaw agent --agent ops --message "Summarize logs"
openclaw agent --session-id 1234 --message "Summarize inbox" --thinking medium
openclaw agent --to +15555550123 --message "Trace logs" --verbose on --json
openclaw agent --to +15555550123 --message "Summon reply" --deliver
openclaw agent --agent ops --message "Generate report" --deliver --reply-channel slack --reply-to "#reports"
```

## Флаги

-   `--local`: запустить локально (требует API-ключи провайдера модели в вашей оболочке)
-   `--deliver`: отправить ответ в выбранный канал
-   `--channel`: канал доставки (`whatsapp|telegram|discord|googlechat|slack|signal|imessage`, по умолчанию: `whatsapp`)
-   `--reply-to`: переопределение цели доставки
-   `--reply-channel`: переопределение канала доставки
-   `--reply-account`: переопределение идентификатора аккаунта доставки
-   `--thinking <off|minimal|low|medium|high|xhigh>`: сохранить уровень мышления (только для моделей GPT-5.2 + Codex)
-   `--verbose <on|full|off>`: сохранить уровень подробного вывода
-   `--timeout <секунды>`: переопределить таймаут агента
-   `--json`: выводить структурированный JSON
