# Вход в браузер

## Ручной вход (рекомендуется)

Когда сайт требует входа, **войдите вручную** в профиль **хост**-браузера (браузер openclaw). **Не** передавайте модели ваши учетные данные. Автоматизированные входы часто запускают анти-бот защиту и могут заблокировать аккаунт. Вернуться к основной документации по браузеру: [Браузер](./browser.md).

## Какой профиль Chrome используется?

OpenClaw управляет **выделенным профилем Chrome** (с именем `openclaw`, интерфейс с оранжевым оттенком). Он отделен от вашего повседневного профиля браузера. Два простых способа получить к нему доступ:

1.  **Попросите агента открыть браузер**, а затем войдите самостоятельно.
2.  **Откройте его через CLI**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

Если у вас несколько профилей, передайте `--browser-profile ` (по умолчанию `openclaw`).

## X/Twitter: рекомендуемый процесс

-   **Чтение/поиск/треды:** используйте **хост**-браузер (ручной вход).
-   **Публикация обновлений:** используйте **хост**-браузер (ручной вход).

## Изоляция (sandbox) + доступ к хост-браузеру

Изолированные сессии браузера **с большей вероятностью** вызовут срабатывание детектора ботов. Для X/Twitter (и других строгих сайтов) предпочтительнее использовать **хост**-браузер. Если агент изолирован, инструмент браузера по умолчанию использует изолированную среду. Чтобы разрешить управление хостом:

```json
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

Затем нацельтесь на хост-браузер:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

Или отключите изоляцию для агента, который публикует обновления.
