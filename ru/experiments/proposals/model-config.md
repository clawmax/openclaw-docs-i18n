# Исследование конфигурации моделей

В этом документе собраны **идеи** для будущей конфигурации моделей. Это не спецификация к реализации. Текущее поведение описано в:

-   [Модели](../../concepts/models.md)
-   [Отказоустойчивость моделей](../../concepts/model-failover.md)
-   [OAuth + профили](../../concepts/oauth.md)

## Мотивация

Операторы хотят:

-   Несколько профилей аутентификации на провайдера (личный и рабочий).
-   Простой выбор `/model` с предсказуемыми запасными вариантами.
-   Чёткое разделение между текстовыми моделями и моделями с поддержкой изображений.

## Возможное направление (общий уровень)

-   Сохранить выбор модели простым: `провайдер/модель` с опциональными псевдонимами.
-   Разрешить провайдерам иметь несколько профилей аутентификации с явным порядком.
-   Использовать глобальный список запасных вариантов, чтобы все сессии переключались на них единообразно.
-   Переопределять маршрутизацию изображений только при явной настройке.
