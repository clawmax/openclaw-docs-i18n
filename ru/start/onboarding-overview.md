# Обзор адаптации

OpenClaw поддерживает несколько путей адаптации в зависимости от того, где работает Шлюз и как вы предпочитаете настраивать провайдеров.

## Выберите путь адаптации

-   **Мастер CLI** для macOS, Linux и Windows (через WSL2).
-   **Приложение для macOS** для управляемого первого запуска на Mac с Apple silicon или Intel.

## Мастер адаптации CLI

Запустите мастер в терминале:

```bash
openclaw onboard
```

Используйте мастер CLI, когда вы хотите получить полный контроль над Шлюзом, рабочей областью, каналами и навыками. Документация:

-   [Мастер адаптации (CLI)](./wizard.md)
-   [Команда `openclaw onboard`](../cli/onboard.md)

## Адаптация через приложение для macOS

Используйте приложение OpenClaw, когда вам нужна полностью управляемая настройка на macOS. Документация:

-   [Адаптация (Приложение для macOS)](./onboarding.md)

## Пользовательский провайдер

Если вам нужна конечная точка, которой нет в списке, включая размещённых провайдеров, предоставляющих стандартные API OpenAI или Anthropic, выберите **Пользовательский провайдер** в мастере CLI. Вам будет предложено:

-   Выбрать совместимый с OpenAI, совместимый с Anthropic или **Неизвестный** (автоопределение).
-   Ввести базовый URL и ключ API (если это требуется провайдером).
-   Указать идентификатор модели и необязательный псевдоним.
-   Выбрать идентификатор конечной точки (Endpoint ID), чтобы несколько пользовательских конечных точек могли сосуществовать.

Подробные шаги описаны в документации по адаптации CLI выше.
