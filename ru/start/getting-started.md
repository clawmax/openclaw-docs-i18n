# Начало работы

Цель: перейти от нуля к первому рабочему чату с минимальной настройкой.

> **ℹ️** Самый быстрый чат: откройте Панель управления (настройка каналов не требуется). Выполните `openclaw dashboard` и общайтесь в браузере, или откройте `http://127.0.0.1:18789/` на хосте шлюза. Документация: [Дашборд](../web/dashboard.md) и [Панель управления](../web/control-ui.md).

## Предварительные требования

-   Node 22 или новее

> **💡** Проверьте версию Node командой `node --version`, если не уверены.

## Быстрая настройка (CLI)

### Шаг 1: Установите OpenClaw (рекомендуется)

> **ℹ️** Другие способы установки и требования: [Установка](../install.md).

### Шаг 2: Запустите мастер начальной настройки

```bash
openclaw onboard --install-daemon
```

Мастер настраивает аутентификацию, параметры шлюза и опциональные каналы. Подробности см. в [Мастер начальной настройки](./wizard.md).

### Шаг 3: Проверьте Шлюз

Если вы установили службу, она уже должна быть запущена:

```bash
openclaw gateway status
```

### Шаг 4: Откройте Панель управления

```bash
openclaw dashboard
```

 

> **✅** Если Панель управления загрузилась, ваш Шлюз готов к использованию.

## Дополнительные проверки и возможности

Полезно для быстрых тестов или отладки.

```bash
openclaw gateway --port 18789
```

Требуется настроенный канал.

```bash
openclaw message send --target +15555550123 --message "Hello from OpenClaw"
```

## Полезные переменные окружения

Если вы запускаете OpenClaw от имени сервисной учетной записи или хотите использовать пользовательские пути для конфигурации/состояния:

-   `OPENCLAW_HOME` задает домашний каталог, используемый для разрешения внутренних путей.
-   `OPENCLAW_STATE_DIR` переопределяет каталог состояния.
-   `OPENCLAW_CONFIG_PATH` переопределяет путь к файлу конфигурации.

Полный справочник по переменным окружения: [Переменные окружения](../help/environment.md).
