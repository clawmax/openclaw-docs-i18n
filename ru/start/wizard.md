# Мастер адаптации (CLI)

Мастер адаптации — это **рекомендуемый** способ настройки OpenClaw на macOS, Linux или Windows (через WSL2; настоятельно рекомендуется). Он настраивает локальный Шлюз или подключение к удаленному Шлюзу, а также каналы, навыки и параметры рабочей области в одном пошаговом процессе.

```bash
openclaw onboard
```

> **ℹ️** Самый быстрый первый чат: откройте Панель управления (настройка канала не требуется). Выполните `openclaw dashboard` и общайтесь в браузере. Документация: [Панель управления](../web/dashboard.md).

 Чтобы перенастроить позже:

```bash
openclaw configure
openclaw agents add <name>
```

> **ℹ️** `--json` не подразумевает неинтерактивный режим. Для скриптов используйте `--non-interactive`.

 

> **💡** Мастер адаптации включает шаг настройки веб-поиска, где вы можете выбрать провайдера (Perplexity, Brave, Gemini, Grok или Kimi) и вставить свой API-ключ, чтобы агент мог использовать `web_search`. Это также можно настроить позже с помощью `openclaw configure --section web`. Документация: [Веб-инструменты](../tools/web.md).

## Быстрый старт vs Расширенный

Мастер начинается с **Быстрого старта** (значения по умолчанию) или **Расширенного** (полный контроль). 

-   Локальный шлюз (loopback)
-   Рабочая область по умолчанию (или существующая)
-   Порт шлюза **18789**
-   Аутентификация шлюза **Токен** (автоматически генерируется, даже на loopback)
-   Политика инструментов по умолчанию для новых локальных настроек: `tools.profile: "coding"` (существующий явный профиль сохраняется)
-   Изоляция ЛС по умолчанию: локальная адаптация записывает `session.dmScope: "per-channel-peer"`, если значение не задано. Подробности: [Справочник по адаптации CLI](./wizard-cli-reference.md#outputs-and-internals)
-   Экспозиция Tailscale **Выключена**
-   Telegram и WhatsApp ЛС по умолчанию используют **список разрешенных** (вас попросят указать номер телефона)

-   Открывает каждый шаг (режим, рабочая область, шлюз, каналы, демон, навыки).

## Что настраивает мастер

**Локальный режим (по умолчанию)** проводит вас через следующие шаги:

1.  **Модель/Аутентификация** — выберите любого поддерживаемого провайдера/поток аутентификации (API-ключ, OAuth или токен настройки), включая Пользовательского провайдера (совместимого с OpenAI, совместимого с Anthropic или Неизвестный с автоопределением). Выберите модель по умолчанию. Примечание по безопасности: если этот агент будет запускать инструменты или обрабатывать контент вебхуков/hooks, предпочтительнее использовать самую мощную модель последнего поколения и держать политику инструментов строгой. Более слабые/старые модели легче подвержены инъекциям промптов. Для неинтерактивных запусков `--secret-input-mode ref` сохраняет ссылки на переменные окружения в профилях аутентификации вместо значений API-ключа в открытом виде. В неинтерактивном режиме `ref` переменная окружения провайдера должна быть установлена; передача флагов ключа в строке без этой переменной окружения завершается с ошибкой. В интерактивных запусках выбор режима ссылки на секрет позволяет указать либо переменную окружения, либо настроенную ссылку на провайдера (`file` или `exec`), с быстрой предварительной проверкой перед сохранением.
2.  **Рабочая область** — Расположение файлов агента (по умолчанию `~/.openclaw/workspace`). Создает начальные файлы.
3.  **Шлюз** — Порт, адрес привязки, режим аутентификации, экспозиция Tailscale. В интерактивном режиме токена выберите хранение токена в открытом виде по умолчанию или перейдите на SecretRef. Путь SecretRef токена для неинтерактивного режима: `--gateway-token-ref-env <ENV_VAR>`.
4.  **Каналы** — WhatsApp, Telegram, Discord, Google Chat, Mattermost, Signal, BlueBubbles или iMessage.
5.  **Демон** — Устанавливает LaunchAgent (macOS) или пользовательский юнит systemd (Linux/WSL2). Если аутентификация по токену требует токен и `gateway.auth.token` управляется через SecretRef, установка демона проверяет его, но не сохраняет разрешенный токен в метаданные среды службы супервизора. Если аутентификация по токену требует токен и настроенная ссылка SecretRef не разрешена, установка демона блокируется с указанием действий. Если настроены и `gateway.auth.token`, и `gateway.auth.password`, а `gateway.auth.mode` не задан, установка демона блокируется до явного задания режима.
6.  **Проверка состояния** — Запускает Шлюз и проверяет его работу.
7.  **Навыки** — Устанавливает рекомендуемые навыки и опциональные зависимости.

> **ℹ️** Повторный запуск мастера **не** стирает ничего, если вы явно не выберете **Сброс** (или не передадите `--reset`). CLI `--reset` по умолчанию сбрасывает конфигурацию, учетные данные и сессии; используйте `--reset-scope full`, чтобы включить рабочую область. Если конфигурация недействительна или содержит устаревшие ключи, мастер предложит сначала запустить `openclaw doctor`.

 **Удаленный режим** настраивает только локальный клиент для подключения к Шлюзу в другом месте. Он **не** устанавливает и не изменяет ничего на удаленном хосте.

## Добавить еще одного агента

Используйте `openclaw agents add `, чтобы создать отдельного агента с собственной рабочей областью, сессиями и профилями аутентификации. Запуск без `--workspace` запускает мастер. Что он устанавливает:

-   `agents.list[].name`
-   `agents.list[].workspace`
-   `agents.list[].agentDir`

Примечания:

-   Рабочие области по умолчанию следуют шаблону `~/.openclaw/workspace-`.
-   Добавьте `bindings` для маршрутизации входящих сообщений (мастер может это сделать).
-   Флаги для неинтерактивного режима: `--model`, `--agent-dir`, `--bind`, `--non-interactive`.
