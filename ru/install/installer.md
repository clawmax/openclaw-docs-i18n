# Внутреннее устройство установщика

OpenClaw поставляется с тремя скриптами установки, доступными по адресу `openclaw.ai`.

| Скрипт | Платформа | Что делает |
| --- | --- | --- |
| [`install.sh`](#installsh) | macOS / Linux / WSL | Устанавливает Node при необходимости, устанавливает OpenClaw через npm (по умолчанию) или git, может запустить первоначальную настройку. |
| [`install-cli.sh`](#install-clish) | macOS / Linux / WSL | Устанавливает Node + OpenClaw в локальный префикс (`~/.openclaw`). Не требует прав root. |
| [`install.ps1`](#installps1) | Windows (PowerShell) | Устанавливает Node при необходимости, устанавливает OpenClaw через npm (по умолчанию) или git, может запустить первоначальную настройку. |

## Быстрые команды

 

> **ℹ️** Если установка прошла успешно, но `openclaw` не находится в новом терминале, см. [Устранение неполадок Node.js](./node.md#troubleshooting).

## install.sh

> **💡** Рекомендуется для большинства интерактивных установок на macOS/Linux/WSL.

### Порядок работы (install.sh)

### Шаг 1: Определение ОС

Поддерживает macOS и Linux (включая WSL). Если обнаружен macOS, устанавливает Homebrew при его отсутствии.

### Шаг 2: Проверка Node.js 22+

Проверяет версию Node и устанавливает Node 22 при необходимости (через Homebrew на macOS, через скрипты NodeSource на Linux apt/dnf/yum).

### Шаг 3: Проверка Git

Устанавливает Git, если он отсутствует.

### Шаг 4: Установка OpenClaw

-   Метод `npm` (по умолчанию): глобальная установка через npm
-   Метод `git`: клонирование/обновление репозитория, установка зависимостей через pnpm, сборка, затем установка обёртки в `~/.local/bin/openclaw`

### Шаг 5: Пост-установочные задачи

-   Запускает `openclaw doctor --non-interactive` при обновлениях и установках через git (по возможности)
-   Пытается запустить первоначальную настройку, когда это уместно (доступен TTY, настройка не отключена, проверки начальной загрузки и конфигурации пройдены)
-   По умолчанию устанавливает `SHARP_IGNORE_GLOBAL_LIBVIPS=1`

### Обнаружение локальной копии исходного кода

Если скрипт запущен внутри локальной копии OpenClaw (присутствуют `package.json` + `pnpm-workspace.yaml`), он предлагает:

-   использовать локальную копию (`git`), или
-   использовать глобальную установку (`npm`)

Если TTY недоступен и метод установки не задан, по умолчанию используется `npm` с предупреждением. Скрипт завершается с кодом `2` при неверном выборе метода или недопустимых значениях `--install-method`.

### Примеры (install.sh)

 

| Флаг | Описание |
| --- | --- |
| `--install-method npm\|git` | Выбор метода установки (по умолчанию: `npm`). Псевдоним: `--method` |
| `--npm` | Сокращение для метода npm |
| `--git` | Сокращение для метода git. Псевдоним: `--github` |
| `--version <version\|dist-tag>` | Версия npm или dist-тег (по умолчанию: `latest`) |
| `--beta` | Использовать beta dist-тег, если доступен, иначе откатиться к `latest` |
| `--git-dir ` | Каталог для клонирования (по умолчанию: `~/openclaw`). Псевдоним: `--dir` |
| `--no-git-update` | Пропустить `git pull` для существующей копии |
| `--no-prompt` | Отключить запросы |
| `--no-onboard` | Пропустить первоначальную настройку |
| `--onboard` | Включить первоначальную настройку |
| `--dry-run` | Вывести действия без их применения |
| `--verbose` | Включить отладочный вывод (`set -x`, логи npm уровня notice) |
| `--help` | Показать справку (`-h`) |

| Переменная | Описание |
| --- | --- |
| `OPENCLAW_INSTALL_METHOD=git\|npm` | Метод установки |
| `OPENCLAW_VERSION=latest\|next\|` | Версия npm или dist-тег |
| `OPENCLAW_BETA=0\|1` | Использовать beta, если доступен |
| `OPENCLAW_GIT_DIR=` | Каталог для клонирования |
| `OPENCLAW_GIT_UPDATE=0\|1` | Включить/отключить обновления git |
| `OPENCLAW_NO_PROMPT=1` | Отключить запросы |
| `OPENCLAW_NO_ONBOARD=1` | Пропустить первоначальную настройку |
| `OPENCLAW_DRY_RUN=1` | Режим пробного запуска |
| `OPENCLAW_VERBOSE=1` | Режим отладки |
| `OPENCLAW_NPM_LOGLEVEL=error\|warn\|notice` | Уровень логирования npm |
| `SHARP_IGNORE_GLOBAL_LIBVIPS=0\|1` | Управление поведением sharp/libvips (по умолчанию: `1`) |

## install-cli.sh

> **ℹ️** Предназначен для сред, где требуется разместить всё в локальном префиксе (по умолчанию `~/.openclaw`) без зависимости от системного Node.

### Порядок работы (install-cli.sh)

### Шаг 1: Установка локальной среды выполнения Node

Загружает архив Node (по умолчанию `22.22.0`) в `/tools/node-v` и проверяет SHA-256.

### Шаг 2: Проверка Git

Если Git отсутствует, пытается установить его через apt/dnf/yum на Linux или Homebrew на macOS.

### Шаг 3: Установка OpenClaw в префикс

Устанавливает через npm с использованием `--prefix `, затем создаёт обёртку в `/bin/openclaw`.

### Примеры (install-cli.sh)

 

| Флаг | Описание |
| --- | --- |
| `--prefix ` | Префикс установки (по умолчанию: `~/.openclaw`) |
| `--version ` | Версия OpenClaw или dist-тег (по умолчанию: `latest`) |
| `--node-version ` | Версия Node (по умолчанию: `22.22.0`) |
| `--json` | Выводить события в формате NDJSON |
| `--onboard` | Запустить `openclaw onboard` после установки |
| `--no-onboard` | Пропустить первоначальную настройку (по умолчанию) |
| `--set-npm-prefix` | На Linux принудительно установить префикс npm в `~/.npm-global`, если текущий префикс недоступен для записи |
| `--help` | Показать справку (`-h`) |

| Переменная | Описание |
| --- | --- |
| `OPENCLAW_PREFIX=` | Префикс установки |
| `OPENCLAW_VERSION=` | Версия OpenClaw или dist-тег |
| `OPENCLAW_NODE_VERSION=` | Версия Node |
| `OPENCLAW_NO_ONBOARD=1` | Пропустить первоначальную настройку |
| `OPENCLAW_NPM_LOGLEVEL=error\|warn\|notice` | Уровень логирования npm |
| `OPENCLAW_GIT_DIR=` | Путь для очистки устаревших данных (используется при удалении старой копии подмодуля `Peekaboo`) |
| `SHARP_IGNORE_GLOBAL_LIBVIPS=0\|1` | Управление поведением sharp/libvips (по умолчанию: `1`) |

## install.ps1

### Порядок работы (install.ps1)

### Шаг 1: Проверка PowerShell и среды Windows

Требуется PowerShell 5+.

### Шаг 2: Проверка Node.js 22+

При отсутствии пытается установить через winget, затем Chocolatey, затем Scoop.

### Шаг 3: Установка OpenClaw

-   Метод `npm` (по умолчанию): глобальная установка через npm с использованием выбранного `-Tag`
-   Метод `git`: клонирование/обновление репозитория, установка/сборка через pnpm и установка обёртки в `%USERPROFILE%\.local\bin\openclaw.cmd`

### Шаг 4: Пост-установочные задачи

Добавляет необходимый каталог bin в пользовательский PATH, когда это возможно, затем запускает `openclaw doctor --non-interactive` при обновлениях и установках через git (по возможности).

### Примеры (install.ps1)

 

| Флаг | Описание |
| --- | --- |
| `-InstallMethod npm\|git` | Метод установки (по умолчанию: `npm`) |
| `-Tag ` | npm dist-тег (по умолчанию: `latest`) |
| `-GitDir ` | Каталог для клонирования (по умолчанию: `%USERPROFILE%\openclaw`) |
| `-NoOnboard` | Пропустить первоначальную настройку |
| `-NoGitUpdate` | Пропустить `git pull` |
| `-DryRun` | Только вывод действий |

| Переменная | Описание |
| --- | --- |
| `OPENCLAW_INSTALL_METHOD=git\|npm` | Метод установки |
| `OPENCLAW_GIT_DIR=` | Каталог для клонирования |
| `OPENCLAW_NO_ONBOARD=1` | Пропустить первоначальную настройку |
| `OPENCLAW_GIT_UPDATE=0` | Отключить git pull |
| `OPENCLAW_DRY_RUN=1` | Режим пробного запуска |

 

> **ℹ️** Если используется `-InstallMethod git` и Git отсутствует, скрипт завершится с выводом ссылки на Git for Windows.

## Устранение неполадок

Git требуется для метода установки `git`. Для установок через `npm` Git всё равно проверяется/устанавливается, чтобы избежать ошибок `spawn git ENOENT`, когда зависимости используют git URL.

В некоторых настройках Linux глобальный префикс npm указывает на пути, принадлежащие root. `install.sh` может переключить префикс на `~/.npm-global` и добавить экспорты PATH в файлы конфигурации оболочки (если эти файлы существуют).

Скрипты по умолчанию устанавливают `SHARP_IGNORE_GLOBAL_LIBVIPS=1`, чтобы избежать сборки sharp с системным libvips. Для переопределения:

```
SHARP_IGNORE_GLOBAL_LIBVIPS=0 curl -fsSL --proto '=https' --tlsv1.2 https://openclaw.ai/install.sh | bash
```

Установите Git for Windows, перезапустите PowerShell, запустите установщик снова.

Выполните `npm config get prefix` и добавьте этот каталог в ваш пользовательский PATH (на Windows суффикс `\bin` не нужен), затем перезапустите PowerShell.

`install.ps1` в настоящее время не предоставляет переключатель `-Verbose`. Используйте трассировку PowerShell для диагностики на уровне скрипта:

```powershell
Set-PSDebug -Trace 1
& ([scriptblock]::Create((iwr -useb https://openclaw.ai/install.ps1))) -NoOnboard
Set-PSDebug -Trace 0
```

Обычно проблема с PATH. См. [Устранение неполадок Node.js](./node.md#troubleshooting).
