# Развертывание на Northflank

Разверните OpenClaw на Northflank с помощью шаблона в один клик и завершите настройку в браузере. Это самый простой путь «без терминала на сервере»: Northflank запускает Шлюз за вас, а вы настраиваете всё через веб-мастер `/setup`.

## Как начать

1.  Нажмите [Deploy OpenClaw](https://northflank.com/stacks/deploy-openclaw), чтобы открыть шаблон.
2.  Создайте [аккаунт на Northflank](https://app.northflank.com/signup), если у вас его ещё нет.
3.  Нажмите **Deploy OpenClaw now**.
4.  Установите необходимую переменную окружения: `SETUP_PASSWORD`.
5.  Нажмите **Deploy stack**, чтобы собрать и запустить шаблон OpenClaw.
6.  Дождитесь завершения развертывания, затем нажмите **View resources**.
7.  Откройте сервис OpenClaw.
8.  Откройте публичный URL OpenClaw и завершите настройку по адресу `/setup`.
9.  Откройте Панель управления по адресу `/openclaw`.

## Что вы получаете

-   Размещённый Шлюз OpenClaw + Панель управления
-   Веб-мастер настройки по адресу `/setup` (без команд терминала)
-   Постоянное хранилище через Northflank Volume (`/data`), чтобы конфигурация/учётные данные/рабочая область сохранялись при повторных развертываниях

## Процесс настройки

1.  Перейдите по адресу `https://<ваш-домен-northflank>/setup` и введите ваш `SETUP_PASSWORD`.
2.  Выберите модель/провайдера аутентификации и вставьте ваш ключ.
3.  (Опционально) Добавьте токены для Telegram/Discord/Slack.
4.  Нажмите **Run setup**.
5.  Откройте Панель управления по адресу `https://<ваш-домен-northflank>/openclaw`

Если для Telegram личных сообщений установлен режим сопряжения, мастер настройки может подтвердить код сопряжения.

## Получение токенов для чатов

### Токен бота Telegram

1.  Напишите `@BotFather` в Telegram
2.  Выполните команду `/newbot`
3.  Скопируйте токен (выглядит как `123456789:AA...`)
4.  Вставьте его в `/setup`

### Токен бота Discord

1.  Перейдите по адресу [https://discord.com/developers/applications](https://discord.com/developers/applications)
2.  **New Application** → выберите имя
3.  **Bot** → **Add Bot**
4.  **Enable MESSAGE CONTENT INTENT** в разделе Bot → Privileged Gateway Intents (обязательно, иначе бот упадёт при запуске)
5.  Скопируйте **Bot Token** и вставьте в `/setup`
6.  Пригласите бота на ваш сервер (OAuth2 URL Generator; области действия: `bot`, `applications.commands`)
