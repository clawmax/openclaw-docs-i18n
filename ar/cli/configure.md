# configure

معالج تفاعلي لإعداد بيانات الاعتماد والأجهزة والإعدادات الافتراضية للعملاء. ملاحظة: يتضمن قسم **النموذج** الآن اختيارًا متعددًا لقائمة `agents.defaults.models` المسموح بها (التي تظهر في `/model` وفي منتقي النماذج). نصيحة: `openclaw config` دون أمر فرعي يفتح نفس المعالج. استخدم `openclaw config get|set|unset` للتعديلات غير التفاعلية. مواضيع ذات صلة:

-   مرجع إعدادات البوابة: [الإعدادات](../gateway/configuration.md)
-   أمر config في واجهة سطر الأوامر: [Config](./config.md)

ملاحظات:

-   اختيار مكان تشغيل البوابة يقوم دائمًا بتحديث `gateway.mode`. يمكنك اختيار "متابعة" دون الأقسام الأخرى إذا كان هذا هو كل ما تحتاجه.
-   الخدمات الموجهة للقنوات (Slack/Discord/Matrix/Microsoft Teams) تطلب قوائم القنوات/الغرف المسموح بها أثناء الإعداد. يمكنك إدخال الأسماء أو المعرفات؛ يحاول المعالج تحويل الأسماء إلى معرفات عندما يكون ذلك ممكنًا.
-   إذا قمت بتشغيل خطوة تثبيت الخدمة الخلفية (daemon)، فإن المصادقة باستخدام الرمز (token) تتطلب رمزًا، ويتم إدارة `gateway.auth.token` عبر SecretRef، يقوم الأمر configure بالتحقق من SecretRef ولكنه لا يحفظ قيم الرمز العادية التي تم حلها في بيانات وصف بيئة خدمة المشرف (supervisor service environment metadata).
-   إذا كانت المصادقة باستخدام الرمز تتطلب رمزًا وكان SecretRef المُهيأ للرمز غير محلول، فإن الأمر configure يمنع تثبيت الخدمة الخلفية ويقدم إرشادات قابلة للتنفيذ.
-   إذا تم تهيئة كل من `gateway.auth.token` و `gateway.auth.password` وكان `gateway.auth.mode` غير مضبوط، فإن الأمر configure يمنع تثبيت الخدمة الخلفية حتى يتم ضبط الوضع بشكل صريح.

## أمثلة

```bash
openclaw configure
openclaw configure --section model --section channels
```
