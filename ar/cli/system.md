# system

مساعدات على مستوى النظام للبوابة: قائمة انتظار أحداث النظام، التحكم في نبضات القلب، وعرض الحضور.

## الأوامر الشائعة

```bash
openclaw system event --text "Check for urgent follow-ups" --mode now
openclaw system heartbeat enable
openclaw system heartbeat last
openclaw system presence
```

## system event

أضف حدث نظام إلى قائمة الانتظار في الجلسة **الرئيسية**. ستقوم نبضة القلب التالية بحقنه كسطر `System:` في المطالبة. استخدم `--mode now` لتحفيز نبضة القلب فورًا؛ `next-heartbeat` تنتظر النبضة المجدولة التالية. الوسائط:

-   `--text `: نص حدث النظام (مطلوب).
-   `--mode `: `now` أو `next-heartbeat` (الافتراضي).
-   `--json`: إخراج قابل للقراءة الآلية.

## system heartbeat last|enable|disable

ضوابط نبضات القلب:

-   `last`: عرض آخر حدث لنبضة القلب.
-   `enable`: إعادة تشغيل نبضات القلب (استخدم هذا إذا تم تعطيلها).
-   `disable`: إيقاف نبضات القلب مؤقتًا.

الوسائط:

-   `--json`: إخراج قابل للقراءة الآلية.

## system presence

سرد إدخالات حضور النظام الحالية التي تعرفها البوابة (العقد، النسخ، وسطور الحالة المماثلة). الوسائط:

-   `--json`: إخراج قابل للقراءة الآلية.

## ملاحظات

-   يتطلب بوابة قيد التشغيل يمكن الوصول إليها عبر التكوين الحالي (محلي أو بعيد).
-   أحداث النظام عابرة ولا يتم استمرارها عبر عمليات إعادة التشغيل.
