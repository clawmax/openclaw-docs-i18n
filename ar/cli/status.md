# status

تشخيصات للقنوات والجلسات.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

ملاحظات:

-   `--deep` يقوم بتشغيل مسح حي (WhatsApp Web + Telegram + Discord + Google Chat + Slack + Signal).
-   يتضمن الإخراج مخازن الجلسات لكل وكيل عند تكوين وكلاء متعددين.
-   تتضمن النظرة العامة حالة تثبيت/تشغيل خدمة Gateway + node host عند توفرها.
-   تتضمن النظرة العامة قناة التحديث + معرف Git SHA (للنسخ المأخوذة من المصدر).
-   تظهر معلومات التحديث في النظرة العامة؛ إذا كان هناك تحديث متاح، يطبع الأمر `status` تلميحًا لتشغيل `openclaw update` (انظر [التحديث](../install/updating.md)).
-   أوامر الحالة للقراءة فقط (`status`, `status --json`, `status --all`) تحل SecretRefs المدعومة لمسارات التكوين المستهدفة عندما يكون ذلك ممكنًا.
-   إذا تم تكوين SecretRef لقناة مدعومة ولكنه غير متاح في مسار الأمر الحالي، تبقى الحالة للقراءة فقط وتُبلغ بإخراج متدهور بدلاً من التعطل. يُظهر الإخراج البشري تحذيرات مثل "الرمز المميز المُكوَّن غير متاح في مسار الأمر هذا"، ويتضمن الإخراج JSON `secretDiagnostics`.
