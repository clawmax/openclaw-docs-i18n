# التثبيت

هل اتبعت بالفعل [الشروع في العمل](./start/getting-started.md)؟ أنت جاهز — هذه الصفحة مخصصة لطرق التثبيت البديلة، والتعليمات الخاصة بمنصة معينة، والصيانة.

## متطلبات النظام

-   **[Node 22+](./install/node.md)** (سيقوم [سكريبت المثبت](#install-methods) بتثبيته إذا كان مفقودًا)
-   نظام macOS، أو Linux، أو Windows
-   `pnpm` فقط إذا كنت تريد البناء من المصدر

> **ℹ️** على نظام Windows، نوصي بشدة بتشغيل OpenClaw تحت [WSL2](https://learn.microsoft.com/en-us/windows/wsl/install).

## طرق التثبيت

> **💡** **سكريبت المثبت** هو الطريقة الموصى بها لتثبيت OpenClaw. فهو يتعامل مع اكتشاف Node، والتثبيت، والتهيئة في خطوة واحدة.

 

> **⚠️** لمضيفات VPS/السحابية، تجنب صور السوق "بنقرة واحدة" من طرف ثالث عند الإمكان. يُفضل استخدام صورة نظام تشغيل أساسية نظيفة (مثل Ubuntu LTS)، ثم قم بتثبيت OpenClaw بنفسك باستخدام سكريبت المثبت.

 

يقوم بتنزيل واجهة سطر الأوامر، وتثبيتها عالميًا عبر npm، وإطلاق معالج التهيئة.

هذا كل شيء — يتعامل السكريبت مع اكتشاف Node، والتثبيت، والتهيئة.لتخطي التهيئة وتثبيت الملف الثنائي فقط:

لجميع الأعلام، متغيرات البيئة، وخيارات CI/الأتمتة، راجع [تفاصيل المثبت الداخلية](./install/installer.md).

إذا كان لديك بالفعل Node 22+ وتفضل إدارة التثبيت بنفسك:

للمساهمين أو أي شخص يريد التشغيل من نسخة محلية.

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install

```

## طرق تثبيت أخرى

## بعد التثبيت

تحقق من أن كل شيء يعمل:

```bash
openclaw doctor         # التحقق من مشاكل التكوين
openclaw status         # حالة البوابة
openclaw dashboard      # فتح واجهة المستخدم في المتصفح
```

إذا كنت بحاجة إلى مسارات وقت تشغيل مخصصة، استخدم:

-   `OPENCLAW_HOME` للمسارات الداخلية القائمة على الدليل الرئيسي
-   `OPENCLAW_STATE_DIR` لموقع الحالة القابلة للتعديل
-   `OPENCLAW_CONFIG_PATH` لموقع ملف التكوين

راجع [متغيرات البيئة](./help/environment.md) للحصول على تفاصيل كاملة وأولويات.

## استكشاف الأخطاء وإصلاحها: openclaw غير موجود

تشخيص سريع:

```bash
node -v
npm -v
npm prefix -g
echo "$PATH"
```

إذا لم يكن `$(npm prefix -g)/bin` (macOS/Linux) أو `$(npm prefix -g)` (Windows) موجودًا في `$PATH` الخاص بك، فإن طرفيتك لا تستطيع العثور على الملفات الثنائية العالمية لـ npm (بما في ذلك `openclaw`).الإصلاح — أضفه إلى ملف بدء تشغيل الطرفية الخاص بك (`~/.zshrc` أو `~/.bashrc`):

```bash
export PATH="$(npm prefix -g)/bin:$PATH"
```

على Windows، أضف ناتج أمر `npm prefix -g` إلى PATH الخاص بك.ثم افتح طرفية جديدة (أو نفذ `rehash` في zsh / `hash -r` في bash).
