# تتبع الاستخدام

## ما هو

-   يسحب استخدام/حصص المزود مباشرة من نقاط نهاية الاستخدام الخاصة بهم.
-   لا توجد تكاليف تقديرية؛ فقط النوافذ المبلغ عنها من قبل المزود.

## أين يظهر

-   `/status` في الدردشات: بطاقة حالة غنية بالرموز التعبيرية مع رموز الجلسة + التكلفة التقديرية (مفتاح API فقط). يظهر استخدام المزود **لمزود النموذج الحالي** عند التوفر.
-   `/usage off|tokens|full` في الدردشات: تذييل استخدام لكل استجابة (يظهر OAuth الرموز فقط).
-   `/usage cost` في الدردشات: ملخص التكلفة المحلي المجمع من سجلات جلسة OpenClaw.
-   CLI: `openclaw status --usage` يطبع تفصيلاً كاملاً لكل مزود.
-   CLI: `openclaw channels list` يطبع لقطة الاستخدام نفسها بجانب إعدادات المزود (استخدم `--no-usage` للتخطي).
-   شريط قائمة macOS: قسم "الاستخدام" تحت السياق (فقط إذا كان متاحًا).

## المزودون + بيانات الاعتماد

-   **Anthropic (Claude)**: رموز OAuth في ملفات تعريف المصادقة.
-   **GitHub Copilot**: رموز OAuth في ملفات تعريف المصادقة.
-   **Gemini CLI**: رموز OAuth في ملفات تعريف المصادقة.
-   **Antigravity**: رموز OAuth في ملفات تعريف المصادقة.
-   **OpenAI Codex**: رموز OAuth في ملفات تعريف المصادقة (يُستخدم accountId عند وجوده).
-   **MiniMax**: مفتاح API (مفتاح خطة البرمجة؛ `MINIMAX_CODE_PLAN_KEY` أو `MINIMAX_API_KEY`)؛ يستخدم نافذة خطة البرمجة لمدة 5 ساعات.
-   **z.ai**: مفتاح API عبر env/config/auth store.

يتم إخفاء الاستخدام إذا لم توجد بيانات اعتماد OAuth/API مطابقة.
