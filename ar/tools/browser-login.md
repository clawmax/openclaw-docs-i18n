# تسجيل الدخول للمتصفح

## تسجيل الدخول اليدوي (موصى به)

عندما يتطلب موقع ما تسجيل الدخول، **قم بتسجيل الدخول يدوياً** في ملف تعريف المتصفح **المضيف** (متصفح openclaw). **لا** تعطي النموذج بيانات اعتمادك. غالباً ما تؤدي عمليات تسجيل الدخول الآلي إلى تشغيل دفاعات مكافحة الروبوتات وقد تؤدي إلى قفل الحساب. العودة إلى وثائق المتصفح الرئيسية: [المتصفح](./browser.md).

## أي ملف تعريف Chrome مستخدم؟

يتحكم OpenClaw في **ملف تعريف Chrome مخصص** (اسمه `openclaw`، واجهة مستخدم ذات لون برتقالي). هذا منفصل عن ملف تعريف متصفحك اليومي. طريقتان سهلتان للوصول إليه:

1.  **اطلب من الوكيل فتح المتصفح** ثم سجل الدخول بنفسك.
2.  **افتحه عبر سطر الأوامر**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

إذا كان لديك ملفات تعريف متعددة، مرر `--browser-profile ` (القيمة الافتراضية هي `openclaw`).

## X/Twitter: التدفق الموصى به

-   **القراءة/البحث/المحادثات:** استخدم متصفح **المضيف** (تسجيل دخول يدوي).
-   **نشر التحديثات:** استخدم متصفح **المضيف** (تسجيل دخول يدوي).

## العزل + الوصول إلى متصفح المضيف

جلسات المتصفح المعزولة **أكثر عرضة** لتشغيل اكتشاف الروبوتات. بالنسبة لـ X/Twitter (والمواقع الصارمة الأخرى)، يُفضل استخدام متصفح **المضيف**. إذا كان الوكيل معزولاً، فإن أداة المتصفح تستخدم العزل افتراضياً. للسماح بالتحكم من المضيف:

```json
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

ثم استهدف متصفح المضيف:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

أو قم بتعطيل العزل للوكيل الذي ينشر التحديثات.
