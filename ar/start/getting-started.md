# بدء العمل

الهدف: الانتقال من الصفر إلى أول محادثة عاملة مع الحد الأدنى من الإعداد.

> **ℹ️** أسرع دردشة: افتح واجهة التحكم (لا حاجة لإعداد قناة). نفّذ `openclaw dashboard` وتحدث في المتصفح، أو افتح `http://127.0.0.1:18789/` على مضيف البوابة. الوثائق: [لوحة التحكم](../web/dashboard.md) و [واجهة التحكم](../web/control-ui.md).

## المتطلبات الأساسية

-   Node 22 أو أحدث

> **💡** تحقق من إصدار Node الخاص بك باستخدام `node --version` إذا كنت غير متأكد.

## الإعداد السريع (CLI)

### الخطوة 1: تثبيت OpenClaw (موصى به)

> **ℹ️** طرق تثبيت أخرى والمتطلبات: [التثبيت](../install.md).

### الخطوة 2: تشغيل معالج الإعداد الأولي

```bash
openclaw onboard --install-daemon
```

يقوم المعالج بتكوين المصادقة وإعدادات البوابة والقنوات الاختيارية. راجع [معالج الإعداد الأولي](./wizard.md) للحصول على التفاصيل.

### الخطوة 3: التحقق من البوابة

إذا قمت بتثبيت الخدمة، يجب أن تكون قيد التشغيل بالفعل:

```bash
openclaw gateway status
```

### الخطوة 4: فتح واجهة التحكم

```bash
openclaw dashboard
```

 

> **✅** إذا تم تحميل واجهة التحكم، فإن بوابتك جاهزة للاستخدام.

## فحوصات وإضافات اختيارية

مفيد للاختبارات السريعة أو استكشاف الأخطاء وإصلاحها.

```bash
openclaw gateway --port 18789
```

يتطلب قناة مكونة.

```bash
openclaw message send --target +15555550123 --message "Hello from OpenClaw"
```

## متغيرات البيئة المفيدة

إذا قمت بتشغيل OpenClaw كحساب خدمة أو أردت مواقع مخصصة للتكوين/الحالة:

-   `OPENCLAW_HOME` يحدد دليل المنزل المستخدم لحل المسارات الداخلية.
-   `OPENCLAW_STATE_DIR` يتجاوز دليل الحالة.
-   `OPENCLAW_CONFIG_PATH` يتجاوز مسار ملف التكوين.

مرجع متغيرات البيئة الكامل: [متغيرات البيئة](../help/environment.md).
