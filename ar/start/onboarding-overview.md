# نظرة عامة على الإعداد

يدعم OpenClaw مسارات إعداد متعددة اعتمادًا على مكان تشغيل البوابة وكيفية تفضيلك لتكوين المزودين.

## اختر مسار الإعداد الخاص بك

-   **معالج سطر الأوامر** لنظام macOS وLinux وWindows (عبر WSL2).
-   **تطبيق macOS** للإعداد الموجه في المرة الأولى على أجهزة Mac ذات معالج Apple silicon أو Intel.

## معالج الإعداد عبر سطر الأوامر

شغّل المعالج في الطرفية:

```bash
openclaw onboard
```

استخدم معالج سطر الأوامر عندما تريد تحكمًا كاملاً في البوابة ومساحة العمل والقنوات والمهارات. الوثائق:

-   [معالج الإعداد (سطر الأوامر)](./wizard.md)
-   [أمر `openclaw onboard`](../cli/onboard.md)

## الإعداد عبر تطبيق macOS

استخدم تطبيق OpenClaw عندما تريد إعدادًا موجهًا بالكامل على نظام macOS. الوثائق:

-   [الإعداد (تطبيق macOS)](./onboarding.md)

## مزود مخصص

إذا كنت بحاجة إلى نقطة نهاية غير مدرجة، بما في ذلك المزودين المستضيفين الذين يعرضون واجهات برمجة تطبيقات OpenAI أو Anthropic القياسية، اختر **مزود مخصص** في معالج سطر الأوامر. سيُطلب منك:

-   اختيار متوافق مع OpenAI، أو متوافق مع Anthropic، أو **غير معروف** (الكشف التلقائي).
-   إدخال عنوان URL أساسي ومفتاح API (إذا كان المزود يتطلب ذلك).
-   تقديم معرف النموذج واسم مستعار اختياري.
-   اختيار معرف نقطة النهاية حتى تتمكن نقاط النهاية المخصصة المتعددة من التعايش.

للخطوات التفصيلية، اتبع وثائق إعداد سطر الأوامر أعلاه.
