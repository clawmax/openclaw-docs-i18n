# نموذج الشبكة

تتدفق معظم العمليات عبر البوابة (`openclaw gateway`)، وهي عملية واحدة طويلة الأمد تمتلك اتصالات القنوات ومستوى التحكم عبر WebSocket.

## القواعد الأساسية

-   يُوصى ببوابة واحدة لكل مضيف. وهي العملية الوحيدة المسموح لها بامتلاك جلسة WhatsApp Web. لروبوتات الإنقاذ أو العزل الصارم، قم بتشغيل بوابات متعددة بملفات تعريف ومنافذ معزولة. راجع [بوابات متعددة](./multiple-gateways.md).
-   الوصول المحلي أولاً: افتراضيًا، يستمع WebSocket الخاص بالبوابة على `ws://127.0.0.1:18789`. يولد المعالج التلقائي رمز بوابة افتراضيًا، حتى للوصول المحلي. للوصول عبر شبكة الذيل، قم بتشغيل `openclaw gateway --bind tailnet --token ...` لأن الرموز مطلوبة للوصلات غير المحلية.
-   تتصل العقد ببوابة WebSocket عبر الشبكة المحلية (LAN)، أو شبكة الذيل، أو SSH حسب الحاجة. جسر TCP القديم لم يعد موصى به.
-   يتم تقديم مضيف اللوحة Canvas بواسطة خادم HTTP التابع للبوابة على **نفس المنفذ** الخاص بالبوابة (الافتراضي `18789`):
    -   `/__openclaw__/canvas/`
    -   `/__openclaw__/a2ui/` عند تكوين `gateway.auth` وربط البوابة خارج النطاق المحلي، تكون هذه المسارات محمية بمصادقة البوابة. تستخدم عملاء العقد عناوين URL للقدرات محدودة النطاق بالعقدة مرتبطة بجلسة WebSocket النشطة الخاصة بهم. راجع [تكوين البوابة](./configuration.md) (`canvasHost`, `gateway`).
-   عادةً ما يكون الاستخدام عن بُعد عبر نفق SSH أو شبكة افتراضية خاصة (VPN) لشبكة الذيل. راجع [الوصول عن بُعد](./remote.md) و [الاكتشاف](./discovery.md).
