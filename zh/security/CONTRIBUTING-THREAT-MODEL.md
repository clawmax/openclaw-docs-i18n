# 贡献威胁模型

感谢您帮助 OpenClaw 变得更安全。此威胁模型是一个动态文档，我们欢迎任何人的贡献——您不需要是安全专家。

## 贡献方式

### 添加威胁

发现我们未涵盖的攻击向量或风险？请在 [openclaw/trust](https://github.com/openclaw/trust/issues) 上创建一个 issue，并用您自己的话描述它。您不需要了解任何框架或填写每个字段——只需描述场景即可。**有助于包含的内容（非必需）：**

-   攻击场景及其可能的利用方式
-   OpenClaw 的哪些部分会受到影响（CLI、网关、通道、ClawHub、MCP 服务器等）
-   您认为其严重性如何（低 / 中 / 高 / 严重）
-   任何相关研究、CVE 或真实世界示例的链接

我们将在审核过程中处理 ATLAS 映射、威胁 ID 和风险评估。如果您想包含这些细节，那很好——但不是必需的。

> **这是为了添加到威胁模型，而不是报告实时漏洞。** 如果您发现了可利用的漏洞，请参阅我们的 [信任页面](https://trust.openclaw.ai) 了解负责任的披露说明。

### 建议缓解措施

对如何应对现有威胁有想法？请创建一个引用该威胁的 issue 或 PR。有用的缓解措施是具体且可操作的——例如，“在网关处实施每个发送者每分钟 10 条消息的速率限制”比“实施速率限制”更好。

### 提出攻击链

攻击链展示了多个威胁如何组合成一个现实的攻击场景。如果您发现一个危险的组合，请描述其步骤以及攻击者如何将它们链接在一起。一个关于攻击在实践中如何展开的简短叙述比正式的模板更有价值。

### 修复或改进现有内容

拼写错误、澄清、过时信息、更好的示例——欢迎提交 PR，无需先创建 issue。

## 我们使用的工具

### MITRE ATLAS

此威胁模型基于 [MITRE ATLAS](https://atlas.mitre.org/)（AI 系统对抗性威胁态势），这是一个专门为 AI/ML 威胁（如提示注入、工具滥用和智能体利用）设计的框架。您不需要了解 ATLAS 即可贡献——我们会在审核过程中将提交内容映射到该框架。

### 威胁 ID

每个威胁都会获得一个像 `T-EXEC-003` 这样的 ID。类别如下：

| 代码 | 类别 |
| --- | --- |
| RECON | 侦察 - 信息收集 |
| ACCESS | 初始访问 - 获得入口 |
| EXEC | 执行 - 运行恶意操作 |
| PERSIST | 持久化 - 维持访问 |
| EVADE | 防御规避 - 避免检测 |
| DISC | 发现 - 了解环境 |
| EXFIL | 数据窃取 |
| IMPACT | 影响 - 损害或破坏 |

ID 由维护者在审核期间分配。您不需要自己选择。

### 风险等级

| 等级 | 含义 |
| --- | --- |
| **严重** | 完全的系统失陷，或高可能性 + 严重影响 |
| **高** | 可能造成重大损害，或中等可能性 + 严重影响 |
| **中** | 中等风险，或低可能性 + 高影响 |
| **低** | 可能性低且影响有限 |

如果您不确定风险等级，只需描述影响，我们会进行评估。

## 审核流程

1.  **分类** - 我们会在 48 小时内审核新的提交
2.  **评估** - 我们验证可行性，分配 ATLAS 映射和威胁 ID，验证风险等级
3.  **文档化** - 我们确保所有内容格式正确且完整
4.  **合并** - 添加到威胁模型和可视化中
